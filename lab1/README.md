##  Лабораторная работа №1

### Вариант 24(Модель умного дома)

Предметная область: автоматизация и управление домашней средой.

Важные сущности: умный дом, умные устройства, системы безопасности, управление освещением, климат-контроль, умные бытовые приборы.

Операции: операция управления освещением и климатом, операция мониторинга систем безопасности, операция управления умными устройствами через мобильное приложение, операция создания расписания автоматизации, операция дистанционного контроля за домом.
##  О программе
Программа представляет собой имитационную модель робота, оснащённого сенсорами, механическими манипуляторами, системой передвижения и программным обеспечением.

### Система способна:
Управлять освещением (включение, выключение, регулировка яркости)

Управлять климат-контролем (установка температуры)

Управлять пылесосом (обычная и влажная уборка)

Управлять чайником (кипячение, нагрев воды)

Обеспечивать безопасность (сигнализация с режимами охраны)

Создавать правила автоматизации по времени или статусу устройств

# Умный дом (Smart Home System)

Система управления умным домом с интерфейсом командной строки.

##  Описание классов

###  SmartHome
**Главный класс-контейнер**, содержащий комнаты, устройства, правила и систему безопасности.

**Функции:**
- Хранит все комнаты (`rooms`)
- Хранит систему безопасности (`security_system`)
- Хранит правила автоматизации (`automation_rules`)
- Хранит устройство хозяина (`owner_device`)
- Управляет созданием комнат
- Поиск комнат и устройств
- Проверка правил автоматизации
- Сохранение/загрузка состояния

###  Room
**Представляет комнату** в доме, содержит устройства.

**Функции:**
- Хранит устройства в комнате
- Добавление/удаление устройств
- Поиск устройств по ID
- Поиск устройств по типу

### 🔌 SmartDevice (ABC)
**Абстрактный базовый класс** для всех устройств.

**Функции:**
- Общие поля: id, name, room_id, status
- Абстрактный метод `get_device_type()`
- Включение/выключение устройства
- Получение статуса

###  LightDevice
**Конкретная реализация** устройства освещения.

**Функции:**
- Регулировка яркости (0-100%)
- Получение статуса с яркостью

###  ClimateDevice
**Конкретная реализация** устройства климат-контроля.

**Функции:**
- Управление температурой (16-30°C)
- Управление влажностью (30-80%)
- Режимы: auto, cool, heat, dry, fan_only
- Скорость вентилятора
- Автоматическая регулировка

###  SmartCleaner
**Конкретная реализация** умного пылесоса.

**Функции:**
- Режимы: обычный, влажная уборка
- Запуск/остановка уборки

###  SmartKettle
**Конкретная реализация** умного чайника.

**Функции:**
- Кипячение воды
- Нагрев до заданной температуры
- Отслеживание уровня воды

###  SecuritySystem
**Конкретная реализация** системы безопасности.

**Функции:**
- Постановка/снятие с охраны
- Запуск тревоги
- Отслеживание состояния

###  OwnerDevice
**Представляет мобильное устройство** пользователя.

**Функции:**
- Подключение/отключение от системы
- Отправка уведомлений
- Отслеживание последнего доступа

###  AutomationRule
**Содержит условие и список действий**, которые выполняются при его истинности.

**Функции:**
- Условие (`condition`) - время или статус устройства
- Действия (`actions`) - список команд
- Проверка условия
- Выполнение действий
- Включение/отключение правила

### SmartHomeCLI
**Отвечает за взаимодействие с пользователем** через командную строку.

**Функции:**
- Отображение меню
- Управление устройствами
- Создание правил
- Просмотр состояния
- Сохранение/загрузка
