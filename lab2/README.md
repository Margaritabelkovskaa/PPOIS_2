# Система управления футболистами

## Описание проекта
Десктопное приложение для учета футболистов с графическим интерфейсом. Позволяет добавлять, искать, удалять записи, а также импортировать/экспортировать данные в XML.

## Функциональные возможности

### Основные операции
- **Добавление** новых футболистов через диалоговое окно
- **Поиск** записей по различным критериям с постраничным выводом
- **Удаление** записей по условиям с подтверждением
- **Просмотр** всех записей в главном окне
- **Альтернативное отображение** в виде дерева (группировка по командам)

### Работа с данными
- Хранение в **SQLite** базе данных
- Импорт/экспорт в **XML** (DOM для записи, SAX для чтения)
- 60+ тестовых записей для демонстрации

### Интерфейс
- Главное окно с таблицей записей
- Меню и панель инструментов (дублирование команд)
- Диалоговые окна для добавления, поиска, удаления
- **Постраничный вывод** с навигацией (первая, последняя, следующая, предыдущая)
- Календарь для выбора даты рождения
  


## Технологии
- **Python 3.13**
- **PyQt5** - графический интерфейс
- **SQLite** - база данных
- **XML** (SAX/DOM парсеры) - обмен данными

## Архитектура MVC

### Model
Model отвечает за данные и бизнес-логику:

- `Player` — структура записи футболиста (ФИО, дата рождения, команда, город, состав, позиция)
- `Database` — работа с базой **SQLite** (CRUD + поиск + удаление + очистка + тестовые данные)
- `XMLHandler` — экспорт в XML через DOM, импорт из XML через SAX

### View
Представления (PyQt6):

- `MainWindow` — главное окно (таблица + пагинация)
- `AddPlayerDialog` — диалог добавления записи
- `SearchDialog` — диалог поиска (условия + результаты + пагинация)
- `DeleteDialog` — диалог удаления (условия)
- `TreeViewDialog` — древовидное представление (группировка по командам)
- `Paginator` — класс для управления постраничным выводом

### Controller
Связывает Model и View:

- `PlayerController` — обрабатывает действия пользователя:
  - добавление записи
  - получение всех записей
  - поиск (с постраничным выводом)
  - удаление (с сообщением о результате)
  - импорт/экспорт XML
  - очистка всех записей
  - добавление тестовых данных

##  Структура базы данных

**Таблица `players`:**

| Поле | Тип SQLite | Тип Python | Описание |
|------|------------|------------|----------|
| id | INTEGER | int | Первичный ключ (автоинкремент) |
| full_name | TEXT | str | ФИО игрока |

## Постраничный вывод

Пагинация реализована в:
- главном окне (список всех записей)
- диалоге поиска (результаты поиска)

Возможности:
- переход на первую/последнюю страницу
- переход на предыдущую/следующую страницу
- выбор количества записей на странице (5/10/20/50/100)
- отображение текущей страницы и общего количества страниц

---

## Поиск

Поиск выполняется в **отдельном диалоговом окне** с группировкой условий:

### Группа 1 (И)
- ФИО содержит (частичное совпадение, без учета регистра)
- дата рождения (точное совпадение)

### Группа 2 (ИЛИ)
- позиция (точное совпадение)
- состав (основной / запасной)

### Группа 3 (ИЛИ)
- команда содержит (частичное совпадение, без учета регистра)
- город содержит (частичное совпадение, без учета регистра)

**Логика:** (Группа 1) И (Группа 2) И (Группа 3)

---

## Удаление

Удаление записей выполняется в **отдельном диалоговом окне**.
Правила и условия совпадают с условиями поиска.

После удаления приложение показывает сообщение:
- сколько записей удалено
- или что записей по условиям не найдено

---

## Импорт и экспорт XML

### Экспорт (DOM)
Сохранение выполняется через стандартный диалог "Save File".
- Экспортируются **все записи** из базы
- Используется DOM (`xml.dom.minidom`)
- Метод: `XMLHandler.write_to_file(players, filename)`

### Импорт (SAX)
Загрузка выполняется через стандартный диалог "Open File".
- Записи из XML **добавляются** в базу данных
- Используется SAX (`xml.sax`)
- Метод: `XMLHandler.read_from_file(filename)`

## Тестовые данные

Приложение содержит функцию генерации 50 тестовых записей:
- случайные ФИО из списков имен, фамилий, отчеств
- случайные даты рождения (1985-2005)
- случайные команды и города
- случайные позиции и составы

## Сценарии демонстрации

### Сценарий 1 — запуск
1. Запустить приложение
2. Открывается главное окно
3. Отображается таблица футболистов (если есть записи) + пагинация

### Сценарий 2 — добавление записи
1. **Правка → Добавить** (или кнопка на панели инструментов)
2. Ввести ФИО, дату рождения, команду, город, выбрать позицию и состав
3. Нажать **OK**
4. Запись появляется в таблице

### Сценарий 3 — поиск
1. **Правка → Поиск** (или кнопка на панели инструментов)
2. Ввести условия поиска (ФИО, дата, позиция, состав, команда, город)
3. Нажать **Найти**
4. Результаты отображаются в таблице + работает пагинация

### Сценарий 4 — удаление
1. **Правка → Удалить** (или кнопка на панели инструментов)
2. Ввести условия удаления
3. Нажать **Удалить**
4. Появляется сообщение о количестве удалённых записей

### Сценарий 5 — экспорт XML
1. **Файл → Сохранить в XML**
2. Выбрать имя файла
3. Получить XML с текущими записями

### Сценарий 6 — импорт XML
1. **Файл → Загрузить из XML**
2. Выбрать XML файл
3. Записи добавляются в базу, отображаются в главном окне

### Сценарий 7 — тестовые данные
1. **Правка → Добавить тестовые данные (50 игроков)**
2. Подтвердить действие
3. В базу добавляются 50 случайных игроков

### Сценарий 8 — древовидное представление
1. **Вид → Древовидное представление**
2. Открывается окно с группировкой игроков по командам
3. Можно разворачивать/сворачивать узлы для просмотра детальной информации
